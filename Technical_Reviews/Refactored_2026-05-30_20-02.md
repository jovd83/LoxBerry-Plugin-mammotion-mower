## Deferred / Out Of Scope (intentional)

1. **Cross-file warning-string contract** (CGI greps the daemon's `MQTT disconnected` prose to suppress noise). Replacing it with a structured marker touches both the log format and the CGI parser and risks changing the user-visible "Last failure" banner behavior — better done as its own change with a LoxBerry-host smoke test. Left as-is (currently functioning).
2. **Live-device verification of the blade-life nested path** — needs a real PyMammotion `report_data` dump; a debug log now flags the suspect case.
3. **Import-time side effects** (the module creates dirs + a rotating log handler on import). Left intact to avoid destabilizing the proven boot ordering; the test works around it via temp dirs + paho stub.
4. **Dependency currency** (`pymammotion>=0.5.45`, `paho-mqtt>=1.6`) — defer to a `modern-dependency-guard` pass, which is part of the pre-ship gate anyway.

## Residual Risks

- `extract_datapoints()` / `COMMAND_MAP` remain coupled to PyMammotion internals (unofficial, fast-moving). The new unit tests pin the *transform* logic (bit-packing, unit conversion) but not the upstream field names — those can still drift silently. The added debug log is the early-warning signal.
- The new `validate.yml` runs lint/compile/JSON/tests but cannot exercise the Perl CGI or a live cloud login.
- Before any GitHub release: run the mandatory **modern-dependency-guard + api-contract-sentinel + dependency advisory scan**, bump the version per SemVer, and move the `Unreleased` CHANGELOG entries under the new version heading.
